# Audio_Record_Play 程序说明

## 1. 程序概述

本工程是基于 STM32F407 的音频录制、播放与音频处理课程设计。系统使用 ES8388 音频编解码芯片完成音频采集和输出，通过 I2S2 全双工 DMA 搬运音频数据，并使用 SDIO + FatFs 将录制结果保存为 SD 卡根目录下的 WAV 文件。

程序支持基础录音、基础播放、播放音效切换、第二段音频录制、双文件混音、变声、回声、慢放和数字滤波等功能。主循环持续执行音频应用任务和 OLED 刷新任务，根据 `PA0`、`PE8`、`PC13` 三个按键切换工作状态。

当前固件调试标记为：

- `dual-mix-v13`

烧录后可在 SD 卡根目录的 `debug.txt` 中查看 `fw=dual-mix-v13`，用于确认当前运行的是本版程序。

## 2. 主要功能

- `PA0`：开始或停止录音。
- `PE8`：开始或停止播放。
- `PC13`：循环切换模式。
- 普通 `FX:0` 到 `FX:5` 模式下，录音保存为 `record.wav`。
- `REC2` 模式下，录音保存为 `mix.wav`，作为混音的第二段音频。
- `FX:3` 混音模式播放时同时读取 `record.wav` 和 `mix.wav`，对两路 PCM 数据进行加权叠加。
- `FX:5` 使用一阶数字低通滤波器处理播放音频。
- OLED 显示录音状态、播放状态和当前音效/录音目标。
- 录音最长 60 秒，超过后程序自动停止录音。
- 调试信息保存到 SD 卡根目录下的 `debug.txt`。

## 3. 硬件与外设配置

### 3.1 主控与音频芯片

- 主控芯片：STM32F407。
- 音频编解码芯片：ES8388。
- ES8388 通过 `I2C1` 配置寄存器。
- ES8388 通过 `I2S2` 与 STM32 交换音频数据。

### 3.2 音频接口

程序中 I2S2 工作在主机发送模式，并开启全双工模式：

- 音频采样率：`16 kHz`。
- 采样位宽：`16 bit`。
- WAV 文件通道数：单声道。
- I2S 传输通道：左右双通道。
- 实际录音输入：默认选择左声道。
- 录音数字增益：`3` 倍，带饱和限幅。

相关 I2S 引脚：

- `PC2`：I2S2_ext_SD。
- `PC3`：I2S2_SD。
- `PB10`：I2S2_CK。
- `PC6`：I2S2_MCK。
- `PB9`：I2S2_WS。

### 3.3 SD 卡接口

程序使用 SDIO 和 FatFs 访问 SD 卡：

- `PC8`：SDIO_D0。
- `PC9`：SDIO_D1。
- `PC10`：SDIO_D2。
- `PC11`：SDIO_D3。
- `PC12`：SDIO_CK。
- `PD2`：SDIO_CMD。

当前 SDIO 总线宽度配置为 `1 bit`。

### 3.4 按键

三个按键均为上拉输入，按下时为低电平。程序对按键做 25 ms 消抖处理。

| 按键引脚 | 功能 |
| --- | --- |
| `PA0` | 开始/停止录音 |
| `PE8` | 开始/停止播放 |
| `PC13` | 切换 `FX` 音效模式和 `REC2` 第二录音模式 |

录音过程中按 `PC13` 不会切换模式，避免录音目标在写文件时变化。

### 3.5 OLED 显示

OLED 使用软件 I2C 驱动：

- `PD10`：OLED SCL。
- `PE13`：OLED SDA。
- OLED 地址：`0x3C`。
- 刷新周期：约 200 ms。

OLED 显示内容包括：

- `REC: ON` 或 `REC: STOP`。
- `PLAY: ON` 或 `PLAY: STOP`。
- `FX:n`：当前播放音效模式。
- `REC2`：当前为第二段录音目标模式，按 `PA0` 会录制 `mix.wav`。
- `ERROR`、`SD:x`、`H:x`：错误状态和 FatFs/HAL 状态码。

| `Core/Src/oled_ui.c` | OLED 初始化、绘制和状态显示 |

## 5. 运行流程

### 5.1 启动初始化流程

程序从 `main()` 开始执行：

1. 调用 `HAL_Init()` 初始化 HAL 库。
2. 调用 `SystemClock_Config()` 配置系统时钟。
3. 初始化 GPIO、DMA、I2C1、I2S2、SDIO 和 FatFs。
4. 调用 `AudioApp_Init()` 初始化音频应用。
5. 调用 `OLED_UI_Init()` 初始化 OLED 显示。
6. 进入主循环，反复执行 `AudioApp_Task()`、`OLED_UI_Task()` 和 `HAL_Delay(1)`。

音频应用初始化时会先给 ES8388 留出上电稳定时间，然后短暂启动 I2S DMA 给 ES8388 提供时钟，再通过 I2C 配置 ES8388 的 ADC、DAC、音量和 I2S 格式。

### 5.2 录音流程

按下 `PA0` 且系统处于空闲状态时，程序开始录音：

1. 挂载 SD 卡文件系统。
2. 根据当前模式选择录音文件：
   - `FX:0` 到 `FX:5`：创建或覆盖 `record.wav`。
   - `REC2`：创建或覆盖 `mix.wav`。
3. 写入临时 WAV 文件头。
4. 清空 I2S 发送、接收和单声道缓冲区。
5. 启动 I2S2 全双工 DMA。
6. DMA 半传输和全传输完成后，在主循环中处理对应半缓冲区。
7. 从 I2S 双通道采样中选择左声道，进行 3 倍数字增益放大和饱和限幅。
8. 将处理后的 16 bit 单声道 PCM 数据写入当前 WAV 文件。
9. 再次按下 `PA0` 或录音达到 60 秒后停止录音。
10. 停止 I2S DMA，更新 WAV 文件头中的文件长度和数据长度，刷新并关闭文件。

停止录音后，`debug.txt` 会记录本次录音写入的文件名：

- `record_file=record.wav`
- `record_file=mix.wav`

### 5.3 播放流程

按下 `PE8` 且系统处于空闲状态时，程序开始播放：

1. 挂载 SD 卡文件系统。
2. 打开 SD 卡根目录下的 `record.wav`。
3. 跳过 44 字节 WAV 文件头。
4. 从 `record.wav` 读取 PCM 数据填充播放缓冲区。
5. 根据当前 `FX` 模式进行音频处理。
6. 将处理后的单声道数据复制到 I2S 左右两个输出通道。
7. 启动 I2S2 全双工 DMA 输出音频。
8. DMA 半传输和全传输完成后继续读取下一段音频。
9. 到达文件末尾或再次按下 `PE8` 后停止播放。

在 `FX:3` 混音模式下，程序会额外尝试打开 `mix.wav`。如果 `mix.wav` 存在，则同时读取两路 WAV 数据；如果不存在或提前结束，第二路按静音处理，不会导致播放错误。

## 6. 模式说明

按下 `PC13` 可以循环切换模式，OLED 会显示当前模式。

| 显示 | 模式 | 说明 |
| --- | --- | --- |
| `FX:0` | 普通播放 | 不改变 `record.wav` 的音频数据 |
| `FX:1` | 变声/变调 | 通过块内重采样、符号调制和限幅放大改变声音听感 |
| `FX:2` | 回声 | 原始声音叠加延迟反馈信号 |
| `FX:3` | 混音 | 同时读取 `record.wav` 和 `mix.wav`，按 `record / 2 + mix / 2` 加权叠加 |
| `FX:4` | 慢放 | 减少文件读取量并重复样本，使播放速度变慢 |
| `FX:5` | 数字滤波 | 一阶低通数字滤波器，使声音更平滑 |
| `REC2` | 第二段录音 | 不是播放音效；此模式下按 `PA0` 会录制 `mix.wav` |

音效只作用于播放过程，不会修改 SD 卡中的 `record.wav` 或 `mix.wav` 文件。

## 7. 推荐使用方法

### 7.1 普通录音和播放

1. 烧录程序并插入 SD 卡。
2. 上电后等待 OLED 初始化完成。
3. 保持 OLED 显示 `FX:0` 或其他非 `REC2` 模式。
4. 按 `PA0` 开始录音，OLED 显示 `REC: ON`。
5. 等约 1 秒后开始说话。
6. 再次按 `PA0` 停止录音，生成 `record.wav`。
7. 按 `PE8` 播放 `record.wav`。

### 7.2 两段音频混音

1. 在 `FX:0` 或其他非 `REC2` 模式下按 `PA0`，录制第一段声音，生成 `record.wav`。
2. 按 `PC13` 多次切换，直到 OLED 底部显示 `REC2`。
3. 在 `REC2` 模式下按 `PA0`，录制第二段声音，生成 `mix.wav`。
4. 再按 `PC13` 切换到 `FX:3`。
5. 按 `PE8` 播放，程序会同时读取 `record.wav` 和 `mix.wav` 并进行混音。

混音算法可以表述为：

```c
output = record_sample / 2 + mix_sample / 2;
```

该算法属于对两路数字音频信号的加权叠加，并使用饱和限幅避免溢出。

### 7.3 数字滤波演示

1. 先录制 `record.wav`。
2. 按 `PC13` 切换到 `FX:5`。
3. 按 `PE8` 播放。
4. 程序对播放样本执行一阶低通数字滤波，使声音变化更平滑。

滤波核心形式为：

```c
y[n] = (3 * y[n - 1] + x[n]) / 4;
```

其中 `x[n]` 是当前输入样本，`y[n]` 是当前滤波输出样本。

## 8. 重要录音注意事项

开始录音后不要立刻说话。建议按下 `PA0` 后先等待 OLED 显示 `REC: ON`，或者心里默数约 1 秒后再开始讲话。

原因是程序开始录音时需要完成 SD 卡文件创建、WAV 文件头写入、I2S DMA 启动和音频缓冲区稳定等过程。如果按键刚按下就马上说话，最开始的一小段声音可能还没有被稳定采集并写入文件。

推荐节奏：

1. 按下 `PA0`。
2. 等 OLED 显示 `REC: ON`，或等待约 1 秒。
3. 开始说话或播放待录制声音。
4. 讲话结束后再次按 `PA0` 停止录音。

## 9. 文件说明

### 9.1 record.wav

`record.wav` 是第一段录音结果文件，位于 SD 卡根目录。文件格式为标准 PCM WAV：

- 文件头长度：44 字节。
- 采样率：16 kHz。
- 位宽：16 bit。
- 通道数：1。
- 最大录音时长：60 秒。

普通 `FX:0` 到 `FX:5` 模式下按 `PA0` 录音会创建或覆盖该文件。

### 9.2 mix.wav

`mix.wav` 是第二段录音结果文件，位于 SD 卡根目录。格式同样为 16 kHz、16 bit、单声道 PCM WAV。

只有在 OLED 显示 `REC2` 时按 `PA0` 录音，程序才会创建或覆盖 `mix.wav`。该文件主要用于 `FX:3` 混音模式。

### 9.3 debug.txt

`debug.txt` 是调试日志文件，位于 SD 卡根目录。程序在录音停止或录音启动失败时写入调试信息，包括：

- 固件标记：`fw=dual-mix-v13`。
- 当前事件。
- 本次录音写入的文件名：`record_file=...`。
- 已录制字节数。
- 录音通道。
- 当前 `FX` 模式。
- HAL 和 FatFs 状态码。
- I2S 状态和错误码。
- 停止录音阶段的 WAV 更新、同步和关闭结果。
- DMA 回调次数。
- 左右声道和单声道采样统计。
- ES8388 关键寄存器值。

当录音异常、无声音或 SD 卡读写失败时，可以查看该文件辅助定位问题。

## 10. 异常情况与排查

- OLED 显示 `ERROR`：说明程序进入错误状态，可根据 OLED 上的 `SD:x`、`H:x` 或 `debug.txt` 判断原因。
- 无法录音：检查 SD 卡是否插好、是否能被 FatFs 挂载、ES8388 是否初始化成功。
- 录音开头缺失：按下 `PA0` 后等待约 1 秒再说话。
- 播放无声音：确认 SD 卡根目录存在有效的 `record.wav`，并检查耳机、扬声器或音频输出连接。
- `FX:3` 没有听到第二段声音：确认已经在 `REC2` 模式下录制过 `mix.wav`，并且 `mix.wav` 位于 SD 卡根目录。
- 声音过小：当前程序已设置 ES8388 输出音量和 3 倍录音数字增益，可检查麦克风输入和硬件连接。
- 声音破音：可能是录音输入太大，3 倍数字增益触发限幅。可以降低说话音量或后续将增益调小。
- 播放音效没有变化：确认不是在录音过程中按 `PC13`，因为录音状态下程序不会切换模式。

## 11. 答辩说明要点

- 录音部分：系统通过 I2S2 全双工 DMA 从 ES8388 采集 16 bit PCM 数据，并通过 FatFs 保存为标准 WAV 文件。
- 增益部分：录音数据写入前进行 3 倍数字增益放大，并使用饱和限幅避免整数溢出。
- 混音部分：系统支持两段独立音频采集，分别保存为 `record.wav` 和 `mix.wav`；播放 `FX:3` 时对两路 PCM 采样进行加权叠加，实现数字混音。
- 滤波部分：`FX:5` 使用一阶低通数字滤波器，公式为 `y[n] = (3y[n-1] + x[n]) / 4`。
- 音效部分：所有音效都在播放阶段实时处理，不会修改原始录音文件。

## 12. 总结

本工程实现了一个较完整的嵌入式音频录放系统。程序使用 I2S DMA 实现实时音频数据搬运，使用 FatFs 保存标准 WAV 文件，并通过 OLED 提供清晰的状态反馈。新版程序增加了 `REC2` 第二段录音模式和 `FX:3` 双文件混音功能，更符合“不同声音混叠”的混音要求，同时保留变声、回声、慢放和数字滤波等课程设计扩展功能。
