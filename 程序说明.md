# Audio_Record_Play 程序说明

## 1. 程序概述

本程序是基于 STM32F407 的音频录制与播放课程设计工程，实现了通过板载按键控制录音、播放和播放音效切换的功能。系统使用 ES8388 音频编解码芯片采集和输出音频，通过 I2S2 全双工 DMA 传输音频数据，并使用 SDIO + FatFs 将录制结果保存到 TF/SD 卡中的 WAV 文件。

程序启动后会初始化 GPIO、DMA、I2C1、I2S2、SDIO、FatFs、ES8388 和 OLED 显示模块。主循环中持续执行音频应用任务和 OLED 刷新任务，根据按键输入切换录音、播放和音效状态。

## 2. 主要功能

- 按下 `PA0` 按键开始或停止录音。
- 按下 `PE8` 按键开始或停止播放。
- 按下 `PC13` 按键切换播放音效模式，录音过程中不会切换音效。
- 录音文件保存为 SD 卡根目录下的 `record.wav`。
- 调试信息保存为 SD 卡根目录下的 `debug.txt`。
- OLED 显示当前录音、播放和音效状态。
- 录音最长 60 秒，超过后程序会自动停止录音。

## 3. 硬件与外设配置

### 3.1 主控与音频芯片

- 主控芯片：STM32F407。
- 音频编解码芯片：ES8388。
- ES8388 通过 `I2C1` 配置寄存器。
- ES8388 通过 `I2S2` 与 STM32 交换音频数据。

### 3.2 音频接口

程序中 I2S2 工作在主机发送模式，并开启全双工模式：

- 音频采样率：`16 kHz`。
- 采样位宽：`16 bit`。
- 录音文件通道数：单声道。
- I2S 传输通道：左右双通道。
- 实际录音通道：默认取左声道。
- 录音数字增益：`3` 倍。

相关引脚由 CubeMX 生成代码配置：

- `PC2`：I2S2_ext_SD。
- `PC3`：I2S2_SD。
- `PB10`：I2S2_CK。
- `PC6`：I2S2_MCK。
- `PB9`：I2S2_WS。

### 3.3 SD 卡接口

程序使用 SDIO 和 FatFs 访问 SD 卡：

- `PC8`：SDIO_D0。
- `PC9`：SDIO_D1。
- `PC10`：SDIO_D2。
- `PC11`：SDIO_D3。
- `PC12`：SDIO_CK。
- `PD2`：SDIO_CMD。

当前 SDIO 初始化中总线宽度配置为 `1 bit`。

### 3.4 按键

三个按键均配置为上拉输入，按下时为低电平：

| 按键引脚 | 功能 |
| --- | --- |
| `PA0` | 开始录音 / 停止录音 |
| `PE8` | 开始播放 / 停止播放 |
| `PC13` | 切换播放音效模式 |

程序对按键做了 25 ms 消抖处理，只有检测到稳定的按下动作后才会执行对应功能。

### 3.5 OLED 显示

OLED 使用软件 I2C 驱动：

- `PD10`：OLED SCL。
- `PE13`：OLED SDA。
- OLED 地址：`0x3C`。
- 刷新周期：约 200 ms。

OLED 会显示：

- `REC: ON` 或 `REC: STOP`。
- `PLAY: ON` 或 `PLAY: STOP`。
- `FX:n`，表示当前播放音效模式。
- 若出现错误，会显示 `ERROR` 以及 SD/FatFs 和 HAL 状态码。

## 4. 程序文件结构

主要用户代码位于 `Core` 目录中：

| 文件 | 作用 |
| --- | --- |
| `Core/Src/main.c` | 系统入口，初始化外设并循环调用音频任务和 OLED 任务 |
| `Core/Src/audio_app.c` | 音频应用核心逻辑，包括按键处理、录音、播放、音效和调试日志 |
| `Core/Inc/audio_app.h` | 音频应用状态和接口声明 |
| `Core/Src/es8388.c` | ES8388 寄存器读写和初始化配置 |
| `Core/Inc/es8388.h` | ES8388 驱动接口声明 |
| `Core/Src/wav.c` | WAV 文件头生成、写入和更新 |
| `Core/Inc/wav.h` | WAV 文件接口声明 |
| `Core/Src/oled_ui.c` | OLED 初始化、绘制和状态显示 |
| `Core/Inc/oled_ui.h` | OLED UI 接口声明 |

CubeMX 生成的外设初始化文件包括 `gpio.c`、`i2c.c`、`i2s.c`、`sdio.c`、`dma.c`、`fatfs.c` 等。

## 5. 程序运行流程

### 5.1 启动初始化流程

程序从 `main()` 函数开始执行：

1. 调用 `HAL_Init()` 初始化 HAL 库。
2. 调用 `SystemClock_Config()` 配置系统时钟。
3. 初始化 GPIO、DMA、I2C1、I2S2、SDIO 和 FatFs。
4. 调用 `AudioApp_Init()` 初始化音频应用。
5. 调用 `OLED_UI_Init()` 初始化 OLED 显示。
6. 进入主循环，反复执行 `AudioApp_Task()`、`OLED_UI_Task()` 和 `HAL_Delay(1)`。

音频应用初始化时会先等待 ES8388 上电稳定，然后启动一次短暂的 I2S DMA 传输，为 ES8388 提供时钟，随后通过 I2C 配置 ES8388 的 ADC、DAC、音量和 I2S 格式。

### 5.2 录音流程

按下 `PA0` 且系统处于空闲状态时，程序开始录音：

1. 挂载 SD 卡文件系统。
2. 创建或覆盖 SD 卡根目录下的 `record.wav`。
3. 先写入一个临时 WAV 文件头。
4. 清空 I2S 发送、接收和单声道缓存。
5. 启动 I2S2 全双工 DMA。
6. DMA 半传输和全传输完成后，在主循环中分别处理半缓冲区数据。
7. 从 I2S 双通道采样中选择左声道，进行 3 倍数字增益放大。
8. 将处理后的 16 bit 单声道 PCM 数据写入 `record.wav`。
9. 再次按下 `PA0` 或录音达到 60 秒后停止录音。
10. 停止 I2S DMA，更新 WAV 文件头中的文件长度和数据长度，刷新并关闭文件。

### 5.3 播放流程

按下 `PE8` 且系统处于空闲状态时，程序开始播放：

1. 挂载 SD 卡文件系统。
2. 打开 SD 卡根目录下的 `record.wav`。
3. 跳过 44 字节 WAV 文件头。
4. 从文件中读取 PCM 数据填充播放缓冲区。
5. 根据当前 `FX` 模式对音频进行处理。
6. 将单声道数据复制到 I2S 左右两个输出通道。
7. 启动 I2S2 全双工 DMA 输出音频。
8. DMA 半传输和全传输完成后继续读取下一段音频。
9. 到达文件末尾或再次按下 `PE8` 后停止播放。

### 5.4 播放音效模式

按下 `PC13` 可以循环切换播放音效模式，OLED 通过 `FX:n` 显示当前模式：

| 显示 | 模式 | 说明 |
| --- | --- | --- |
| `FX:0` | 普通播放 | 不改变音频数据 |
| `FX:1` | 变调 | 通过块内重采样和符号调制改变声音听感 |
| `FX:2` | 回声 | 原始声音叠加延迟反馈 |
| `FX:3` | 混音 | 原始声音叠加两路延迟声音 |
| `FX:4` | 慢速 | 减少文件读取量并重复样本，使播放速度变慢 |
| `FX:5` | 滤波 | 一阶低通滤波，使声音更平滑 |

音效只作用于播放过程，不会改变 SD 卡中的 `record.wav` 文件。

## 6. 使用方法

1. 将程序编译并下载到 STM32F407 开发板。
2. 插入可正常识别的 TF/SD 卡。
3. 上电后等待 OLED 显示初始化完成。
4. 按下 `PA0` 开始录音，OLED 显示 `REC: ON`。
5. 再次按下 `PA0` 停止录音，程序会生成或更新 `record.wav`。
6. 按下 `PC13` 选择需要的播放音效模式。
7. 按下 `PE8` 播放 `record.wav`，OLED 显示 `PLAY: ON`。
8. 播放过程中再次按下 `PE8` 可以提前停止播放。

## 7. 重要录音注意事项

开始录制后不要立刻说话，建议按下 `PA0` 后先等待一下，看到 OLED 显示 `REC: ON` 或等待约 1 秒后再开始说话。

原因是程序在开始录音时需要完成 SD 卡文件创建、WAV 文件头写入、I2S DMA 启动和音频缓冲区稳定等过程。如果按下录音键后马上说话，最开始的一小段声音可能还没有被稳定采集并写入文件，容易出现开头音频丢失或录不进去的情况。

实际使用时推荐操作节奏为：

1. 按下 `PA0`。
2. 等 OLED 显示 `REC: ON`，或心里默数约 1 秒。
3. 再开始讲话或播放待录制声音。
4. 讲话结束后再按 `PA0` 停止录音。

## 8. 文件说明

### 8.1 record.wav

`record.wav` 是录音结果文件，位于 SD 卡根目录。文件格式为标准 PCM WAV：

- 文件头长度：44 字节。
- 采样率：16 kHz。
- 位宽：16 bit。
- 通道数：1。
- 最大录音时长：60 秒。

每次重新开始录音时，程序会以覆盖方式创建 `record.wav`，旧录音会被新录音替换。

### 8.2 debug.txt

`debug.txt` 是调试日志文件，位于 SD 卡根目录。程序在录音停止或录音启动失败时写入调试信息，包括：

- 固件标记。
- 当前事件。
- 已录制字节数。
- 录音通道。
- 当前音效模式。
- HAL 和 FatFs 状态码。
- I2S 状态和错误码。
- DMA 回调次数。
- 左右声道和单声道采样统计。
- ES8388 关键寄存器值。

当录音异常、没有声音或 SD 卡读写失败时，可以查看该文件辅助定位问题。

## 9. 异常情况与排查

- OLED 显示 `ERROR`：说明程序进入错误状态，可根据 OLED 上的 SD/HAL 状态码或 `debug.txt` 判断原因。
- 无法录音：检查 SD 卡是否插好、是否能被 FatFs 挂载、ES8388 是否初始化成功。
- 录音开头缺失：按下 `PA0` 后等待约 1 秒再说话。
- 播放无声音：确认 SD 卡根目录存在有效的 `record.wav`，并检查耳机、扬声器或音频输出连接。
- 声音过小：当前程序已经设置了 ES8388 输出音量和 3 倍录音数字增益，可检查麦克风输入和硬件连接。
- 播放音效没有变化：确认不是在录音过程中按 `PC13`，因为录音状态下程序不会切换音效模式。

## 10. 总结

本工程完成了一个较完整的嵌入式音频录放系统。程序使用 I2S DMA 实现实时音频数据搬运，使用 FatFs 保存标准 WAV 文件，并通过 OLED 提供清晰的状态反馈。录音、播放和音效切换逻辑集中在 `audio_app.c` 中，结构清晰，便于后续扩展更多音频处理算法或增加更复杂的人机交互功能。
